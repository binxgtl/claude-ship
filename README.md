> Ship Claude-generated projects to GitHub in seconds — parse, scaffold, and publish with one command.

Tired of manually setting up GitHub repositories and scaffolding projects generated by AI? `claude-ship` automates this tedious process. It intelligently parses your Claude-generated output, scaffolds a complete project structure, and publishes it directly to a new GitHub repository, all with a single, intuitive command. Whether you're iterating on AI-assisted code or kicking off new projects, `claude-ship` accelerates your development workflow from idea to deployed repository.

## ✨ Features

-   **AI-Powered Parsing**: Understands and extracts project structure, code files, and README content from raw AI model outputs.
-   **Intelligent Scaffolding**: Creates a robust project directory with essential files, respecting best practices for various tech stacks.
-   **GitHub Integration**: Seamlessly creates new GitHub repositories, optionally making them private, and pushes your scaffolded code.
-   **Automatic README Generation**: Leverages AI to generate a comprehensive README.md file based on your project's description, tech stack, and file contents.
-   **Customizable READMEs**: Control the detail level and language (including Vietnamese) of your AI-generated READMEs.
-   **Flexible Configuration**: Use environment variables, a command-line interface, or a dedicated configuration file for seamless integration into your workflow.
-   **Multiple AI Providers**: Supports both Anthropic's Claude (via API key) and Google's Gemini (free tier) for README generation.
-   **Git Integration**: Utilizes `simple-git` for robust local Git operations and pushing to remote repositories.
-   **Dry Run Mode**: Preview all actions without making any actual changes to your file system or GitHub.
-   **Configuration Management**: Save API keys, GitHub tokens, and default settings for a streamlined experience.
-   **File Exclusion**: Excludes common development artifacts and large directories (like `node_modules` or `dist`) during parsing and scaffolding.
-   **Gitignore Presets**: Automatically applies relevant `.gitignore` rules based on detected project technologies.
-   **SSH/HTTPS Remote Support**: Configure whether to use SSH or HTTPS for Git remotes.
-   **Customizable Project Name and Description**: Define your project's identity directly via CLI flags.
-   **Optional README Generation**: Skip AI-generated READMEs and use a basic template if preferred.
-   **Branch Name Customization**: Specify the default Git branch name (e.g., `main` or `master`).
-   **Organization Support**: Create GitHub repositories within a specified organization.
-   **Error Handling**: Provides clear error messages for common issues like missing tokens or invalid input.
-   **Secure Key Storage**: Encrypts sensitive API keys and tokens stored locally for enhanced security.

## 🏗 Architecture

`claude-ship` is a Node.js command-line application designed for efficient project bootstrapping. It orchestrates file parsing, AI-driven content generation, and Git operations. The core logic resides in `src/index.ts`, which utilizes `commander.js` to manage CLI arguments and subcommands. Key modules handle parsing AI output (`src/parser.ts`), detecting project technologies (`src/detector.ts`), generating READMEs (`src/readme.ts`), scaffolding files (`src/scaffold.ts`), interacting with GitHub (`src/github.ts`), and managing configuration (`src/config.ts`).

| Module/File         | Detailed Role                                                                                                                                    |
| :------------------ | :----------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/index.ts`      | The entry point of the CLI application. Initializes and parses arguments using `commander.js`.                                                     |
| `src/cli.ts`        | Defines the `commander.js` program instance, registers subcommands (`ship`, `readme`, `push`, `config`, `name`), and sets up global options.     |
| `src/config.ts`     | Handles loading, saving, and encrypting/decrypting user configuration, including API keys and GitHub tokens, from `~/.claudeship/config.json`.     |
| `src/ui.ts`         | Provides a consistent UI/UX for the CLI, including banners, colored output, and helper functions for chalk.                                       |
| `src/parser.ts`     | Parses raw text input (presumably from an AI) into structured file data (`ParsedFile` and `ParseResult`), identifying code blocks and languages. |
| `src/detector.ts`   | Analyzes project files and their content to detect the underlying tech stack (frameworks, languages, package manager, gitignore presets).        |
| `src/readme.ts`     | Constructs prompts for AI providers (Anthropic, Gemini) to generate README.md content based on project details and desired detail level.          |
| `src/github.ts`     | Manages interactions with the GitHub API, including token resolution, repository creation, and user authentication validation.                       |
| `src/github-oauth.ts` | Implements OAuth device flow for GitHub authentication as an alternative to personal access tokens.                                             |
| `src/scaffold.ts`   | Writes parsed and generated files to the local file system, creates necessary directories, and resolves output paths.                            |
| `src/types.ts`      | Defines shared TypeScript interfaces and types used across the project, such as `ParsedFile`, `TechStack`, `Provider`, and `ReadmeDetail`.       |
| `src/config-ui.ts`  | Provides a command-line interface for interactive configuration, allowing users to set AI keys, GitHub details, and default preferences.          |

## 🔄 How It Works

1.  **Initialization**: The `claude-ship` CLI is invoked via `npx @binxgodteli/claude-ship`. `src/index.ts` creates a `commander.js` program instance.
2.  **Command Parsing**: `commander.js` parses the command-line arguments, identifying the subcommand (e.g., `ship`) and its associated options.
3.  **`ship` Command Execution**:
    *   **Token Resolution**: The command attempts to resolve a GitHub Personal Access Token (PAT) with the following priority: `--token` flag, `gh auth token` (GitHub CLI), `GITHUB_TOKEN` environment variable, or a saved token in `~/.claudeship/config.json`. If none are found, an error is thrown.
    *   **Input Processing**: If a `--file` option is provided, `src/scaffold.ts` reads its content. Otherwise, the input is assumed to be piped or implicitly provided.
    *   **AI Output Parsing**: `src/parser.ts` analyzes the input text, extracting code blocks and identifying potential file paths and content.
    *   **Technology Detection**: `src/detector.ts` inspects the parsed files and their content to infer the project's tech stack (e.g., Node.js, React, Python). This influences `.gitignore` presets and README generation.
    *   **README Generation (Optional)**: If `--no-readme` is not specified, `src/readme.ts` constructs a detailed prompt for the chosen AI provider (Anthropic or Gemini), including project details, detected stack, and file contents. The AI's response is processed as the README content.
    *   **Scaffolding**: `src/scaffold.ts` creates the project directory (defaulting to `./<project-name>`) and writes all parsed and generated files, including the README, respecting any `.gitignore` presets determined by the detector.
    *   **GitHub Repository Creation**: Using the resolved GitHub token, `src/github.ts` creates a new repository on GitHub, setting its name, description, and privacy status as specified by CLI options or saved defaults.
    *   **Git Initialization and Push**: `src/git.ts` initializes a Git repository in the output directory, adds all scaffolded files, commits them, and pushes the commit to the newly created GitHub remote, unless `--no-push` is specified.
4.  **`push` Command Execution**:
    *   This subcommand pushes an **existing local project** to GitHub — no Claude output needed. It creates the GitHub repo if it doesn't exist, optionally regenerates the README with AI, commits everything, and pushes.
5.  **`readme` Command Execution**:
    *   This subcommand focuses on regenerating the README for an existing project located in a specified directory (`--dir`). It uses `src/parser.ts` to read files, `src/detector.ts` to identify the tech stack, and `src/readme.ts` to generate a new README with AI, allowing options for language (`--vi`) and detail level.
6.  **`config` Command Execution**:
    *   This subcommand launches an interactive UI (`src/config-ui.ts`) to manage application settings. Users can securely input and save API keys for AI providers, GitHub Personal Access Tokens, default project settings (like privacy, org, branch), and file exclusion patterns. Sensitive data is encrypted using machine-specific keys.
7.  **`name` Command Execution**:
    *   This subcommand allows setting a default repository name for future `ship` operations.

## 🚀 Getting Started

### Prerequisites

*   **Node.js**: Version 18 or higher. You can check your version by running `node -v` in your terminal.
*   **npm**: Included with Node.js.
*   **GitHub Account**: Required for creating repositories.
*   **GitHub CLI (Optional but Recommended)**: For easier authentication. Install from [cli.github.com](https://cli.github.com/).
*   **API Key for AI Provider (Optional)**: If you wish to use Anthropic's Claude or Google's Gemini for README generation and do not want to rely on saved defaults, you will need an API key.

### Installation

```bash
# Run directly without installing (recommended for one-off use)
npx @binxgodteli/claude-ship --help

# Install globally once for convenience
npm install -g @binxgodteli/claude-ship

# Or add to a project locally
npm install @binxgodteli/claude-ship

# After global installation, run it directly:
claude-ship --help
```

### For contributors

```bash
# Clone the repository
git clone https://github.com/binxgtl/claude-ship.git
cd claude-ship

# Install dependencies
npm install

# Build the project (compiles TypeScript to JavaScript)
npm run build

# Run the CLI for development (using tsx for direct TS execution)
npm run dev -- ship --help
# Or use the built version:
node dist/index.js --help
```

## 📖 Full Usage Reference

`claude-ship` offers a powerful command-line interface to manage your project shipping process.

### The `ship` Command (Default)

Parse a Claude AI response, scaffold a project, and publish it to GitHub in one step. Can be run as `claude-ship ship` or simply `claude-ship` (default command).

**Basic Usage**:

```bash
# Ship a project from a file, with a custom repo name and description
npx @binxgodteli/claude-ship ship --file ./claude-output.txt --name my-cool-project --desc "A project generated by Claude."

# Ship with a private repository
npx @binxgodteli/claude-ship ship --file ./claude-output.txt --name my-private-project --private

# Ship using Gemini provider with your API key
npx @binxgodteli/claude-ship ship --file ./claude-output.txt --provider gemini --api-key YOUR_GEMINI_API_KEY --name gemini-project
```

**Options**:

| Flag | Description |
| :--- | :---------- |
| `--file <path>` | Path to a text file containing the Claude response |
| `--token <token>` | GitHub personal access token |
| `--name <name>` | Project / repository name |
| `--desc <description>` | Short project description |
| `--private` | Create a private GitHub repository |
| `--no-readme` | Skip AI README generation (use built-in template) |
| `--vi` | Generate README in Vietnamese (native developer style) |
| `--provider <name>` | AI provider: `"anthropic"` or `"gemini"` |
| `--api-key <key>` | API key for the selected AI provider |
| `--out <dir>` | Output directory (default: `./<project-name>`) |
| `--no-push` | Scaffold files locally, skip GitHub push |
| `--org <org>` | GitHub organization to create the repo under |
| `--branch <name>` | Git branch name (default: `main`) |
| `--detail <level>` | README detail level: `short`, `normal`, `large`, `carefully` |
| `--max-tokens <n>` | Max output tokens for README generation. `0` = no limit |
| `-d, --dry-run` | Preview what would be created — no writes, no API calls |

**Example with dry-run**:

```bash
npx @binxgodteli/claude-ship ship --file ./claude-output.txt --name preview-project -d
```

---

### The `push` Command

Push an **existing local project** to GitHub — no Claude output needed. Creates the GitHub repo if it doesn't exist, optionally regenerates the README with AI, then commits and pushes everything.

**Basic Usage**:

```bash
# Push the current directory to GitHub (keeps existing README as-is)
npx @binxgodteli/claude-ship push --no-readme

# Push a specific directory with a custom name and description
npx @binxgodteli/claude-ship push --dir ./my-project --name my-repo --desc "My existing project" --no-readme

# Push and let AI regenerate the README
npx @binxgodteli/claude-ship push --provider gemini --api-key YOUR_KEY

# Push to a private repo under an organization
npx @binxgodteli/claude-ship push --org my-org --private --no-readme
```

**Options**:

| Flag | Description |
| :--- | :---------- |
| `--dir <path>` | Project directory to push (default: current working directory) |
| `--name <name>` | Repo name (default: from `package.json` or folder name) |
| `--desc <description>` | Repo description (default: from `package.json`) |
| `--private` | Create as private repo |
| `--token <token>` | GitHub personal access token |
| `--org <org>` | GitHub organization (overrides saved default) |
| `--branch <name>` | Branch name (default: `main`, or saved default) |
| `--no-readme` | Skip README regeneration (use existing) |
| `--vi` | Generate README in Vietnamese |
| `--provider <name>` | AI provider: `"anthropic"` or `"gemini"` |
| `--api-key <key>` | API key for the selected provider |
| `--detail <level>` | README detail level: `short`, `normal`, `large`, `carefully` |
| `--max-tokens <n>` | Max output tokens for README generation. `0` = no limit |
| `--message <msg>` | Git commit message (default: `🚀 Update via claude-ship`) |

---

### The `readme` Command

Regenerate the README for an existing project.

**Basic Usage**:

```bash
# Regenerate README in the current directory using the default provider (anthropic)
npx @binxgodteli/claude-ship readme

# Regenerate README in a specific directory, using Gemini and Vietnamese
npx @binxgodteli/claude-ship readme --dir ../my-existing-project --provider gemini --vi --api-key YOUR_GEMINI_API_KEY
```

**Options**:

| Flag | Description |
| :--- | :---------- |
| `--vi` | Generate in Vietnamese (native dev style) |
| `--dir <path>` | Project directory (default: current working directory) |
| `--provider <name>` | AI provider: `"anthropic"` (default) or `"gemini"` |
| `--api-key <key>` | API key for the selected AI provider |
| `--detail <level>` | README detail level: `short`, `normal`, `large`, `carefully` |
| `--max-tokens <n>` | Max output tokens for README generation. `0` = no limit |
| `--license <type>` | Specify project license type (e.g., MIT, Apache-2.0) |
| `--author <name>` | Author name for copyright line |
| `--github-username <name>` | GitHub username for star history chart URL |
| `--no-screenshot` | Skip adding the screenshot placeholder |

---

### The `config` Command

Launch an interactive configuration UI to set up API keys, GitHub tokens, and default preferences.

```bash
npx @binxgodteli/claude-ship config
```

This command will guide you through several sections:
*   **AI Keys**: Save your Anthropic and Gemini API keys.
*   **GitHub**: Save your GitHub Personal Access Token or initiate OAuth flow.
*   **Defaults**: Set default project privacy, organization, branch name, Git patterns, etc.
*   **README**: Configure default README detail level, license, author, and Vietnamese preference.
*   **Files**: Specify global file/directory exclusion patterns for parsing and scaffolding.

---

### The `name` Command

Set a default repository name to be used by the `ship` command when `--name` is not explicitly provided.

```bash
# Set default name to 'my-default-repo'
npx @binxgodteli/claude-ship name my-default-repo

# Unset the default name
npx @binxgodteli/claude-ship name ""
```

## ⚙️ Configuration Reference

`claude-ship` can be configured using a combination of command-line flags, environment variables, and a configuration file stored at `~/.claudeship/config.json`. Sensitive information like API keys and GitHub tokens are encrypted.

### Environment Variables

| Variable             | Description                                                                                                   |
| :------------------- | :------------------------------------------------------------------------------------------------------------ |
| `GITHUB_TOKEN`       | GitHub Personal Access Token (PAT). Priority: `--token` flag → `gh CLI` → this variable → saved config. |
| `GH_TOKEN`           | Alias for `GITHUB_TOKEN`. |
| `CLAUDE_SHIP_CLIENT_ID` | GitHub OAuth App client ID for device flow authentication. |

### Configuration File (`~/.claudeship/config.json`)

Managed via `claude-ship config`. Sensitive fields are AES-256-CBC encrypted.

**Sensitive Fields (Encrypted)**:

| Field | Description |
| :---- | :---------- |
| `anthropicApiKey` | Your Anthropic API key |
| `geminiApiKey` | Your Google Gemini API key |
| `githubToken` | Your GitHub Personal Access Token (PAT) |

**Plain Fields**:

| Field | Description | Default |
| :---- | :---------- | :------ |
| `defaultProvider` | Default AI provider (`"anthropic"` or `"gemini"`) | `"anthropic"` |
| `defaultPrivate` | Default repo visibility | `false` |
| `githubUsername` | Your GitHub username (used for star history URLs) | `""` |
| `defaultOrg` | Default GitHub organization | `""` |
| `defaultBranch` | Default Git branch name | `"main"` |
| `useSshRemote` | Use SSH URLs for Git remotes instead of HTTPS | `false` |
| `defaultReadmeDetail` | Default README detail level | `"normal"` |
| `defaultLicense` | Default license type | `"MIT"` |
| `projectAuthor` | Default author name for copyright notices | `""` |
| `defaultVi` | Generate READMEs in Vietnamese by default | `false` |
| `maxReadmeTokens` | Max tokens for AI README generation (`0` = no limit) | `0` |
| `gitIncludePatterns` | Glob patterns for files to include in Git commits | `[]` |
| `gitExcludePatterns` | Glob patterns for files to exclude from Git commits | `[]` |
| `aiExcludePatterns` | Glob patterns for files excluded from AI APIs | `[]` |
| `readmeExcludePatterns` | Glob patterns for files excluded from README context | `[]` |

## 🐛 Troubleshooting

*   **"No GitHub token found" Error**:
    1.  Run `gh auth login` and follow the prompts.
    2.  Set the `GITHUB_TOKEN` environment variable.
    3.  Run `claude-ship config` and enter your token or use OAuth.
    4.  Pass your token directly using the `--token` flag.
*   **AI API Key Errors**: Ensure your API keys are correctly entered via `claude-ship config` or `--api-key`. Check your provider's dashboard for key validity and billing status.
*   **Incorrect File Parsing**: Use `--dry-run` to inspect parsed output. Ensure code blocks in your Claude response are clearly delineated with fenced code blocks.
*   **GitHub Repository Creation Failed**: Verify your token has the `repo` scope. Ensure the repo name isn't already taken. If using `--org`, confirm you have the required permissions.
